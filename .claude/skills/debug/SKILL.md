# DEBUG — 小白排错助手

东西坏了不要慌，跟着我一步步排查。

## 当用户说"XX出问题了"或 /debug 时

### Step 1: 收集信息（问3个问题）

1. **出了什么问题？**（截图、报错信息、或者用自己的话描述）
2. **之前能用吗？**（一直坏 / 刚才还好的）
3. **最近改了什么？**（改了代码 / 装了东西 / 什么都没改）

### Step 2: 分类问题

| 问题类型 | 表现 | 排查方向 |
|----------|------|----------|
| **页面白屏/打不开** | 浏览器显示空白或错误 | 前端代码、构建、路由 |
| **样式不对** | 能打开但长得不对 | CSS、Tailwind、响应式 |
| **功能不工作** | 点了没反应 | JavaScript、API调用、事件绑定 |
| **API报错** | 后端返回错误 | 服务器代码、数据库、权限 |
| **部署失败** | 上线不成功 | 环境变量、构建命令、依赖 |
| **数据库问题** | 数据丢了/存不进去 | 连接、表结构、权限 |
| **性能问题** | 太慢了 | 查询优化、缓存、资源大小 |

### Step 3: 排查（我来做，你看着）

```
1. 读错误日志/报错信息
2. 用 Explore 子智能体定位相关代码
3. 分析可能原因（通常列出2-3个）
4. 从最可能的原因开始验证
5. 找到根因
```

### Step 4: 修复
- 告诉你问题出在哪（用人话解释）
- 修复代码
- 用 Playwright 验证修复效果
- 提交 Git（方便回滚）

### Step 5: 教你防范
- 这个问题为什么会出现？
- 下次怎么避免？
- 一句话总结经验

## 常见问题速查

| 你说... | 可能的原因 |
|---------|-----------|
| "页面空白了" | 构建失败 / JS报错 / 路由错误 |
| "样式乱了" | CSS类名写错 / Tailwind没加载 / 缓存 |
| "按钮点了没反应" | 事件没绑定 / API地址错 / 控制台报错 |
| "部署失败了" | 环境变量没设 / 依赖没装 / 构建命令错 |
| "数据不见了" | 数据库连接断了 / 表被清了 / 查询条件错 |

## 原则
- **不慌** — 任何问题都有原因，找到就能修
- **说人话** — 用你能懂的方式解释原因
- **不只修bug** — 还要告诉你为什么，帮你成长
- **先备份** — 修之前先确保有 Git 记录，方便回滚
