# HANDOFF — 项目交接 & 进度保存

每次对话结束前，把当前状态记录下来，下次打开就能无缝继续。

## 当用户说 /handoff 或对话即将结束时

### 自动生成交接文档

检查本次对话做了什么，生成以下内容：

```markdown
# 项目交接记录
> 日期：YYYY-MM-DD
> 会话摘要：[一句话描述这次做了什么]

## 已完成的工作
- [x] 任务1
- [x] 任务2
- [x] 任务3

## 未完成的工作
- [ ] 任务A — 卡在哪了/下一步是什么
- [ ] 任务B — 还没开始，优先级高

## 当前项目状态
- 分支：[当前Git分支]
- 最新提交：[commit hash和消息]
- 可运行吗：[是/否]
- 待修复的问题：[如果有]

## 关键决策记录
- 决策1：选了XX方案，因为XX
- 决策2：放弃了XX，因为XX

## 下次对话的建议开场白
> "上次我们做到了XX，接下来要做XX。请先看 HANDOFF.md 恢复上下文。"

## 重要文件清单
- `文件1` — 做了什么改动
- `文件2` — 做了什么改动
```

### 保存位置
- 保存为项目根目录下的 `HANDOFF.md`
- 每次覆盖（只保留最新状态）
- 同时提交到 Git

## 什么时候该用 /handoff

| 场景 | 触发 |
|------|------|
| 今天干完了 | 保存进度，明天继续 |
| 要换电脑/设备 | 另一边打开就能接上 |
| 对话快到上限了 | 保存后开新会话 |
| 遇到大问题需要思考 | 先存档，想好了再来 |
| 项目告一段落 | 记录里程碑状态 |

## 原则
- **自动化** — 不需要你记，我帮你记
- **可恢复** — 下次打开就能立刻进入状态
- **有重点** — 不记流水账，只记关键信息
