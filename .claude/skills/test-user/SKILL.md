# TEST-USER — 用户视角测试员

不是检查代码对不对，而是**像一个真实用户一样去用你的产品**。

## 核心理念

> 代码审查检查"代码写得对不对"
> 用户测试检查"用户用得爽不爽"
> 这是两件完全不同的事。

## 当用户说 /test-user 或需要测试时

### Step 1: 定义用户角色

先确定要模拟什么样的用户：

```markdown
测试用户画像：
- 身份：[新手用户 / 老用户 / 付费用户 / 管理员]
- 目标：[用户来你的网站想做什么？]
- 设备：[手机 / 电脑 / 平板]
- 耐心程度：[很急 / 正常 / 很有耐心]
```

### Step 2: 设计用户旅程测试

按真实用户的操作路径，用 **Playwright MCP** 模拟操作：

#### 测试场景模板

**场景A：新用户首次访问**
```
1. 打开首页 → 3秒内加载完吗？
2. 能看懂这个网站是干什么的吗？（看标题和副标题）
3. 知道下一步该做什么吗？（CTA按钮明显吗？）
4. 点击CTA → 跳转对了吗？
5. 在手机上看 → 排版正常吗？文字够大吗？
```

**场景B：注册/登录流程**
```
1. 找到注册入口 → 容易找吗？
2. 填写表单 → 字段清楚吗？有提示吗？
3. 故意填错 → 错误提示友好吗？
4. 正确提交 → 多久能完成？跳转对了吗？
5. 刷新页面 → 还是登录状态吗？
```

**场景C：核心功能使用**
```
1. 找到核心功能入口 → 需要几次点击？
2. 使用功能 → 操作直觉吗？需要看说明吗？
3. 等待结果 → 有加载提示吗？
4. 查看结果 → 清楚吗？有用吗？
5. 再试一次 → 流程顺畅吗？
```

**场景D：异常情况**
```
1. 断网了 → 有提示吗？不会白屏吧？
2. 输入奇怪的东西 → 会崩溃吗？
3. 疯狂点击按钮 → 会重复提交吗？
4. 按返回键 → 会丢失数据吗？
5. 很长时间不操作 → 会怎样？
```

**场景E：手机端专项**
```
1. 竖屏看 → 排版正常？
2. 横屏看 → 不会乱？
3. 手指点击 → 按钮够大吗？（最小44px）
4. 滚动 → 流畅吗？
5. 输入框 → 键盘弹出后不会遮住吗？
```

### Step 3: 用 Playwright 执行测试

对每个场景，我会：
```
1. 用 Playwright 打开网页
2. 按用户旅程逐步操作（点击、填写、滚动）
3. 每一步截图记录
4. 检查响应时间
5. 模拟手机/电脑不同设备
6. 模拟网络慢的情况
```

### Step 4: 生成用户体验报告

```markdown

- 测试日期：YYYY-MM-DD
- 测试设备：桌面1920x1080 + 手机375x812
- 模拟用户：新手用户

### 体验评分

- 问题描述 + 截图 + 建议修复
```

## 测试频率建议

| 时机 | 测试什么 |
|------|---------|
| 功能开发完 | 该功能的完整用户旅程 |
| 上线前 | 所有核心场景 + 手机端 |
| 上线后第一周 | 每天快速跑一遍核心流程 |
| 每次大改动后 | 受影响的场景 |

## 原则
- **假装自己啥都不懂** — 不要用开发者思维，用小白思维
- **3秒法则** — 用户3秒内看不懂就会离开
- **3次点击法则** — 核心功能不应该超过3次点击
- **不解释就能用** — 如果需要看说明书，UI就是失败的
