# 一人公司建站全流程手册
# Website Build Playbook — From Idea to Launch

> 这份手册定义了：你做什么、我（Claude）做什么、子智能体做什么
> 目标：一个技术新手，靠AI骑手，从零做出一个专业网站

---

## 铁律：三步不可跳过

```
⚠️ 铁律 1: 不写PRD，不写代码 → 先用 /prd 反向提问，生成需求书+思维导图
⚠️ 铁律 2: 不查清单，不开工   → 再用 /checklist 确认API/账号/素材齐全
⚠️ 铁律 3: 不确认，不往下走   → 每个关键节点等你说"OK"才继续
```

## 标准启动流程（每个新项目必走）

```
你说："我想做一个XXX"
    │
    ▼
/idea — 验证想法（5分钟）
    │
    ▼
/prd — 反向提问提取需求（10-20分钟）
    │  我问你答，一步步理清需求
    │  生成思维导图 → 你确认逻辑
    │  生成用户流程图 → 你确认路径
    │  生成完整PRD文档 → 你确认范围
    │
    ▼
/checklist — 准备清单（10分钟）
    │  API密钥、账号、域名、素材...
    │  全部打勾 ✅ 才能往下走
    │
    ▼
/build — 开始开发
```

## 角色分工表

### 你（老板）的职责 — 只做决策，不写代码

| 阶段 | 你需要做的 | 怎么做 |
|------|-----------|--------|
| 构思 | 说清楚你想做什么、给谁用 | 用 `/idea` 验证想法 |
| 设计 | 选你喜欢的风格和方案 | 看我给的选项，选A/B/C |
| 开发 | 反馈哪里不对、哪里要改 | 看预览截图，说"这里改一下" |
| 测试 | 点点看好不好用 | 打开网址自己试 |
| 上线 | 确认可以发布 | 说"上线吧" |
| 运营 | 分享、推广、收集反馈 | 用 `/content` 写推广内容 |

### Claude 主智能体（我）的职责 — CTO + 项目经理

| 职责 | 说明 |
|------|------|
| **理解需求** | 把你的想法翻译成技术方案 |
| **技术选型** | 选最简单合适的框架和工具 |
| **写代码** | 前端、后端、数据库全栈开发 |
| **调度子智能体** | 把任务分配给专门的子智能体并行处理 |
| **质量把控** | 确保代码安全、性能OK、手机适配 |
| **部署上线** | 配置域名、部署到服务器 |

### 子智能体分工

| 子智能体 | 角色 | 干什么 | 什么时候出动 |
|----------|------|--------|-------------|
| **Explore** | 侦察兵 | 快速搜索代码库、找文件、分析结构 | 项目大了需要找东西时 |
| **Plan** | 架构师 | 设计方案、评估风险、规划步骤 | 开始新功能前做规划 |
| **General Purpose** | 全能工兵 | 执行具体开发任务、调研技术 | 并行处理多个独立任务 |

### MCP 工具分工

| MCP Server | 角色 | 干什么 | 场景举例 |
|------------|------|--------|----------|
| **Playwright** | 测试员 | 打开浏览器看网站效果、自动点击测试 | "帮我看看手机上显示对不对" |
| **Context7** | 图书管理员 | 实时查最新框架文档，防止写过时代码 | 用React/Next.js最新语法 |
| **PostgreSQL** | 数据库管家 | 建表、查数据、管理用户信息 | 用户注册、数据存储 |
| **GitHub** | 仓库管理员 | 提交代码、管理版本、发起PR | 每次改动都有记录 |

## 完整建站流程（8个阶段）

### 阶段 1：构思 💡
**时间：30分钟 | 你主导，我辅助**

```
你说："我想做一个 XXX 网站"
    │
    ▼
/idea — 验证想法（5分钟）
    │ 评估需求、难度、变现、竞争、杠杆
    ▼
/compete — 看看竞品（10分钟）
    │ 找到3-5个类似产品，分析差异化
    ▼
确定方向："做！" 或 "换一个"
    │
    ▼
/pitch — 写一句话定位
    "我是___，为___提供___，解决___问题"
```

**你做的事**：
- [ ] 说出你的想法
- [ ] 决定做还是不做
- [ ] 确认一句话定位

**我做的事**：
- 运行 /idea 评估
- 搜索竞品信息
- 帮你提炼定位

---

### 阶段 2：需求定义 📋 ⚠️ 不可跳过
**时间：15-30分钟 | 我反向提问，你回答**

```
/prd — 启动反向提问
    │
    ├── 第一轮：核心三问
    │   Q1: 你要做什么？
    │   Q2: 给谁用？
    │   Q3: 做到什么程度算完成？
    │
    ├── 第二轮：功能深挖
    │   每个功能追问：触发方式？成功结果？失败处理？
    │   直到你说"够了，就这些"
    │
    ├── 第三轮：非功能需求
    │   设计风格？设备？性能？安全？
    │
    ├── 第四轮：生成思维导图
    │   /mermaid → 画出项目结构
    │   你确认："逻辑对" 或 "这里改一下"
    │
    ├── 第五轮：生成PRD文档
    │   完整的项目需求书（保存为文件）
    │
    └── 第六轮：生成用户流程图
        /mermaid → 画出用户使用路径
        你确认："流程OK"
```

**你做的事**：
- [ ] 回答我的提问（我一次只问一个，不难）
- [ ] 看思维导图确认逻辑
- [ ] 看流程图确认路径
- [ ] 最终确认"需求OK，开始准备"

**我做的事**：
- 反向提问提取你脑中的需求
- 生成思维导图让你一眼看清全貌
- 生成用户流程图确认逻辑
- 输出完整PRD文档

---

### 阶段 3：准备清单 ✅ ⚠️ 不可跳过
**时间：10-30分钟 | 我列清单，你准备材料**

```
/checklist — 根据PRD生成准备清单
    │
    ├── 账号和密钥 🔑
    │   GitHub / 部署平台 / 数据库 / API密钥 / 支付接口
    │
    ├── 素材 🎨
    │   Logo / 配色 / 文案 / 截图
    │
    ├── 技术准备 ⚙️
    │   环境 / MCP配置 / .env模板
    │
    └── 业务准备 📋
        用户画像 / 竞品分析 / 定价方案
    │
    ▼
每项标记状态：✅已有 / ⏳需要做 / ❓需要帮助
    │
    ▼
全部 ✅ → 开工！
有 ❓ → 我帮你搞定
```

**你做的事**：
- [ ] 逐项检查清单
- [ ] 准备需要的账号/素材
- [ ] 不会的标 ❓，我来帮你

**我做的事**：
- 根据PRD生成精确的准备清单
- 帮你搞定标 ❓ 的项目
- 确认所有必需项都是 ✅

---

### 阶段 4：技术设计 📐
**时间：30分钟 | 我主导，你拍板**

```
Plan 子智能体出动
    │
    ▼
选择技术方案（我推荐，你选）
    │ 方案A：纯静态（最简单）
    │ 方案B：Next.js全栈（功能多）
    │ 方案C：Streamlit（数据导向）
    ▼
设计数据库结构
    │ /mermaid → ER图
    ▼
设计API接口
    │ 列出所有接口清单
    ▼
画出系统架构图（用 /mermaid）
    │
    ▼
你确认："就这么做"
```

**你做的事**：
- [ ] 选择技术方案（A/B/C）
- [ ] 确认架构图
- [ ] 同意开始开发

**我做的事**：
- 调度 Plan 子智能体设计架构
- 生成技术方案对比
- 画出 Mermaid 架构图、ER图

---

### 阶段 5：设计 🎨
**时间：1小时 | 我执行，你挑选**

```
确定视觉风格
    │ 科技风 / 简约风 / 活泼风 / 商务风
    ▼
确定配色方案
    │ 我给3套配色，你选一套
    ▼
确定布局模式
    │ 我给参考截图，你说喜欢哪个
    ▼
设计首页 Hero 区域
    │ Playwright 截图预览 → 你看效果
    ▼
你确认："风格OK"
```

**你做的事**：
- [ ] 选视觉风格
- [ ] 选配色
- [ ] 看截图确认效果

**我做的事**：
- 生成多套设计方案
- 用 Playwright 截图给你看效果
- 根据反馈调整

---

### 阶段 6：开发 ⚡
**时间：2-8小时（看复杂度）| 我全权负责**

```
创建项目骨架（/build）
    │
    ▼
并行开发（多个子智能体同时工作）
    │
    ├── General Purpose #1 → 开发首页
    ├── General Purpose #2 → 开发其他页面
    ├── General Purpose #3 → 搭建API后端
    │
    ▼
Context7 → 查最新框架文档，确保代码正确
    │
    ▼
PostgreSQL MCP → 设计数据库表结构
    │
    ▼
代码整合 → 前后端联调
    │
    ▼
GitHub → 提交代码，创建版本记录
```

**你做的事**：
- [ ] 喝咖啡/做其他事 ☕
- [ ] 偶尔看看进度（我会汇报）
- [ ] 回答我可能有的问题

**我做的事**：
- 调度多个子智能体并行开发
- 用 Context7 确保用最新最佳实践
- 设计数据库结构
- 写前端 + 后端 + API
- 提交 Git 记录

### 阶段 7：测试 🧪
**时间：1小时 | 我测试，你验收**

```

    ├── 检查按钮/表单能不能用

    └── 截图对比

    │
    ▼

    │
    ▼
你自己点点看
    │ "这里不对" → 我修
    │ "LGTM 没问题" → 进入下一步
```

**你做的事**：
- [ ] 打开预览链接自己试用
- [ ] 告诉我哪里要改（截图或描述）
- [ ] 确认"可以了"

**我做的事**：
- 用 Playwright 做自动化测试
- 测试多种设备尺寸
- 修复所有发现的Bug
- 做性能优化

## 子智能体调度策略

### 什么时候用几个子智能体？

| 项目复杂度 | 子智能体数量 | 分工 |
|-----------|-------------|------|
| **简单（落地页）** | 0-1个 | 我自己搞定 |
| **中等（多页面网站）** | 2-3个 | 一个做前端，一个做后端，一个做测试 |
| **复杂（SaaS应用）** | 3-5个 | 前端×2 + 后端 + 数据库 + 测试 |

### 子智能体协作流程
```
我（主智能体）接到你的需求
    │
    ├─→ 派 Plan 子智能体设计方案
    │   └─→ 返回：技术方案 + 文件结构 + 任务清单
    │
    ├─→ 派 Explore 子智能体调研现有代码
    │   └─→ 返回：相关文件列表 + 代码分析
    │
    ├─→ 派 General Purpose 子智能体 #1 开发页面A
    │   └─→ 返回：完成的代码
    │
    ├─→ 派 General Purpose 子智能体 #2 开发页面B（并行）
    │   └─→ 返回：完成的代码
    │
    ├─→ 用 Playwright MCP 测试所有页面
    │   └─→ 返回：测试结果 + 截图
    │
    └─→ 整合所有结果 → 提交 GitHub → 汇报给你
```
