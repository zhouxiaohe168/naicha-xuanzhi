# /sprint — 项目冲刺跟进

## 触发方式
用户输入 `/sprint` 时执行此 skill。

## 功能定位
快速掌握项目当前状态，明确本周/本阶段的优先事项，防止精力分散。

---

## 执行流程

### Step 1 — 扫描项目现状
读取以下信息：
- `CLAUDE.md` — 项目背景和规则
- `memory/` 目录 — 历史进度记录
- git log (如有) — 最近提交情况
- 关键文件时间戳 — 判断哪些部分最近被修改

### Step 2 — 输出冲刺看板

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🚀 SPRINT 看板 — [项目名] — [日期]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【本阶段目标】
  → [1-2句话说清这个迭代要达到什么里程碑]

【已完成 ✅】
  ✅ [功能1] — [完成日期/版本]
  ✅ [功能2] — [完成日期/版本]

【进行中 🔄】
  🔄 [功能3] — [当前状态，卡在哪里]
  🔄 [功能4] — [当前状态]

【待开始 📋】
  📋 P1 [高优先级功能] — [原因]
  📋 P2 [中优先级功能] — [原因]
  📋 P3 [低优先级功能] — 可推后

【已知 Bug 🐛】
  🐛 [Bug描述] — [严重程度: 阻塞/影响体验/小问题]

【本次建议下一步 ⚡】
  1. [最重要的一件事，今天就做]
  2. [第二件事]
  3. [可选]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 3 — 优先级判断规则
按以下顺序排优先级：
1. **阻塞用户付费路径** — 最高优先，立即修
2. **影响核心功能可用性** — 高优先，本周内
3. **用户体验优化** — 中优先，有空就做
4. **新功能扩展** — 低优先，稳定后再加
5. **技术债/重构** — 最低，别急

### Step 4 — 询问用户确认
```
你想先攻哪个？我帮你拆成 2-5 分钟小任务。
```
