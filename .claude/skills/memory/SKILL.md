# MEMORY — 一人公司知识银行（灵感来自 Supermemory）

让我越用越懂你。不靠外部服务，就靠一个 Markdown 文件持续积累。

## 核心理念

> Supermemory 用API和数据库实现AI记忆。
> 我们用一个 `MEMORY_BANK.md` 文件实现同样的效果。
> 简单、免费、你能直接看和改。

## 记忆文件结构

存储在项目根目录 `MEMORY_BANK.md`：

```markdown
# 知识银行 (Memory Bank)
> 最后更新：YYYY-MM-DD

## 关于你（自动维护的用户画像）
- 技术水平：新手，正在学习Python/React
- 偏好风格：深色主题、简约设计、科技感
- 偏好技术栈：Python + FastAPI 后端，React + Tailwind 前端
- 决策风格：喜欢看选项对比，然后快速决定
- 沟通偏好：中文，简单直接，不要太多术语

## 项目历史
### 项目1：神盾计划（进行中）
- 类型：加密货币量化交易系统
- 技术栈：Python/FastAPI + React/Vite + ML
- 关键决策：[决策记录]
- 经验教训：[学到了什么]

### 项目2：XXX
- ...

## 技术经验库
### 踩过的坑
- [日期] 问题描述 → 解决方案 → 教训
- [日期] ...

### 有效的模式
```

## 当用户说 /memory 时

### /memory 记录 [内容]
把用户说的内容添加到 MEMORY_BANK.md 的对应分类：

```
用户："记录 今天发现Tailwind的dark:前缀很好用"
→ 添加到"技术经验库/有效的模式"
```

### /memory 查询 [关键词]
搜索 MEMORY_BANK.md 中的相关内容：

```
用户："查询 上次部署用的什么平台"
→ 搜索并返回相关记录
```

### /memory 回顾
展示知识银行的概览和统计：
- 总共记录了多少条经验
- 最近新增了什么
- 有哪些高频出现的模式

### /memory 更新画像
根据最近的对话，自动更新"关于你"部分的用户画像。

## 自动记录触发（我会主动做）

| 场景 | 自动记录 |
|------|---------|
| 用户做了一个技术决策 | 记录到"项目历史/关键决策" |
| 踩了一个坑并解决 | 记录到"技术经验库/踩过的坑" |
| 发现一个好用的工具/方法 | 记录到"技术经验库/有效的模式" |
| 用户表达了偏好 | 更新"偏好设定" |
| 项目完成/里程碑 | 更新"项目历史" |
| /review 复盘时 | 提取关键教训记录 |

## /memory 进化（灵感来自 Everything Claude Code 的 /evolve）

### /memory 进化
分析知识银行中的所有记录，发现规律和模式：

```
分析内容：
1. 踩过的坑有没有重复出现的？→ 生成"防坑规则"
2. 有效的模式能不能提炼成标准做法？→ 更新工作流
3. 偏好有没有变化？→ 更新用户画像
4. 哪些技术栈用得最多最顺？→ 优化推荐
5. 时间都花在哪了？→ 优化效率建议
```

输出进化报告：
```markdown
## 进化报告 — YYYY-MM-DD
### 发现的模式
- 模式1：你经常在XX方面踩坑 → 建议：以后自动检查XX
- 模式2：你最喜欢用XX方案 → 建议：设为默认推荐

### 建议更新
- 更新 /build 默认技术栈为 XX
- 更新 /checklist 新增 XX 检查项
- 更新用户画像：XX

### 成长轨迹
- 上月你是：[水平描述]
- 这月你是：[水平描述]
- 进步了：[具体进步]
```

## 原则
- **自动积累** — 我在对话中自动捕捉值得记录的信息
- **你可控** — 文件是纯Markdown，你能直接看和编辑
- **隐私安全** — 不依赖外部服务，数据在你自己的仓库里
- **越用越强** — 记录越多，我的建议就越精准
- **主动进化** — 不只记录，还会分析规律、优化建议
