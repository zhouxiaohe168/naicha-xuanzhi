# REVIEW — 一人公司复盘系统

帮助用户建立持续复盘的习惯，一人公司最怕的是"瞎忙"。

## 当用户说 /review 时

### 判断复盘类型

**如果是"日复盘"或没有指定**，使用每日复盘模板。
**如果是"周复盘"**，使用每周复盘模板。

---

### 每日复盘（5分钟完成）

引导用户回答以下问题，用对话方式逐个问：

**1. 今天做了什么？**
> 列出今天完成的事情（不用多，1-3件就好）

**2. 最有价值的一件事是什么？**
> 哪件事最接近"赚钱"或"积累资产"？

**3. 今天踩了什么坑？**
> 遇到什么问题？浪费时间在哪了？

**4. 明天最重要的一件事是什么？**
> 如果明天只能做一件事，做什么？

然后帮用户整理成简洁的日志格式：
```
## 日期：YYYY-MM-DD
- 完成：xxx
- 价值点：xxx
- 踩坑：xxx
- 明日重点：xxx
```

---

### 每周复盘（15分钟完成）

引导用户回答以下问题：

**1. 本周成果盘点**
> 这周完成了哪些事？列出所有成果

**2. 飞轮检查**
```
技术实力：本周有提升吗？做了什么？
内容输出：本周发布了什么内容？数据如何？
品牌建设：粉丝/关注者变化？有新的连接吗？
收入情况：本周有收入吗？离目标多远？
```

**3. 时间审计**
> 这周的时间花在了对的地方吗？
> 有没有花太多时间在不产生价值的事上？

**4. 下周三件事**
> 下周最重要的三件事是什么？按优先级排列

整理成周报格式：
```
## 周报：第X周（MM/DD - MM/DD）

### 本周成果
- ...

### 飞轮进展
- 技术：...
- 内容：...
- 品牌：...
- 收入：...

### 经验教训
- ...

### 下周计划
1. ...
2. ...
3. ...
```

## 复盘原则

- **诚实** — 不要自我欺骗，没做就是没做
- **不自责** — 复盘是为了进步，不是为了内耗
- **找规律** — 反复出现的问题才值得重视
- **聚焦行动** — 每次复盘必须产出下一步行动
