# PLAN-TASKS — 任务拆解器（灵感来自 Superpowers/writing-plans）

把一个大任务拆成一堆小任务，每个只要2-5分钟，像拼乐高一样一块块搭起来。

## 当用户说 /plan-tasks 或我要开始开发一个功能时

### Step 1: 理解任务范围

确认：
- 要做什么？（从PRD中提取，或用户直接描述）
- 涉及哪些文件？（用 Explore 子智能体扫描）
- 有什么依赖？（需要先完成什么？）

### Step 2: 生成任务清单

每个任务必须包含：

```markdown
### 任务 X.Y: [任务名称]
- **做什么**: 一句话描述
- **改哪个文件**: `具体/文件/路径.tsx`
- **怎么验证**: 做完后怎么确认是对的
- **预计时间**: 2-5分钟
```

### Step 3: 任务清单格式

```markdown
# 开发计划：[功能名称]
> 总任务数：X | 预计总时间：X分钟

## 阶段 1：基础搭建
- [ ] 1.1 创建 `src/components/Login.tsx` — 基础组件框架 (3min)
- [ ] 1.2 创建 `src/api/auth.ts` — API调用函数 (3min)
      验证：文件存在，无语法错误

## 阶段 2：核心逻辑
- [ ] 2.1 实现登录表单UI (5min)
- [ ] 2.2 连接API调用 (3min)
- [ ] 2.3 处理登录成功/失败 (3min)
      验证：可以输入用户名密码并提交

## 阶段 3：完善
- [ ] 3.1 添加表单验证 (3min)
- [ ] 3.2 添加加载状态 (2min)
- [ ] 3.3 错误提示样式 (2min)
      验证：所有交互状态正常

## 人工检查点 🔍
→ 阶段1完成后：检查文件结构对不对
→ 阶段2完成后：核心功能能用吗？
→ 全部完成后：整体验收
```

### Step 4: 执行模式

**自动批量执行 + 检查点暂停**：
- 同一阶段内的任务：我连续自动完成，不打扰你
- 阶段之间：暂停，汇报进度，等你确认再继续
- 遇到问题：立即停下来跟你沟通

```
阶段1 → [自动执行] → ✅ 汇报 → 你确认 →
阶段2 → [自动执行] → ✅ 汇报 → 你确认 →
阶段3 → [自动执行] → ✅ 汇报 → 最终验收
```

## 拆解原则

1. **每个任务独立可验证** — 做完一个就能测一个
2. **任务之间有明确顺序** — 不会出现"先做后面才能做前面"
3. **人工检查点在关键节点** — 不是每一步都打扰你，但关键时刻要确认
4. **预估时间要诚实** — 超过5分钟就要继续拆分
5. **失败的任务不跳过** — 当前任务没通过验证，不往下走
