# AUTOMATE — 一人公司自动化引擎

一个人的时间有限，把重复的事交给机器，把时间留给创造。

## 当用户说 /automate 时

### Step 1: 理解需求
问用户：
1. **你想自动化什么？**（具体描述那个重复的动作）
2. **多久做一次？**（每天/每周/触发式）
3. **现在手动做要多久？**（评估值不值得自动化）

### 值不值得自动化的判断：
```
手动耗时 × 频率 × 52周 = 年度总耗时

> 10小时/年 → 值得自动化
< 10小时/年 → 可能不值得，除非很烦人
```

### Step 2: 推荐方案

按技术难度从低到高：

| 难度 | 方案 | 适合场景 | 成本 |
|------|------|----------|------|
| 零代码 | **Zapier / Make.com** | 连接SaaS工具（邮件→表格→通知） | 免费-$20/月 |
| 零代码 | **IFTTT** | 简单的"如果…就…"规则 | 免费 |
| 低代码 | **n8n（自部署）** | 复杂工作流，免费无限制 | 免费（需服务器） |
| 低代码 | **GitHub Actions** | 代码相关的定时任务 | 免费 |
| 写代码 | **Python脚本 + Cron** | 完全自定义的自动化 | 免费 |
| 写代码 | **Claude Code + Hooks** | AI驱动的自动化流程 | Claude订阅费 |

### Step 3: 实现自动化

**如果是零代码方案**：给出具体的设置步骤，一步步截图级教程

**如果需要写代码**：直接写出代码，包含：
- 中文注释
- 错误处理（失败了通知用户）
- 日志记录（知道跑没跑成功）
- 部署说明

### Step 4: 监控建议
自动化之后最怕的是"悄悄坏了你不知道"：
- 设置失败告警（邮件/消息通知）
- 定期检查运行日志
- 每月回顾一次自动化流程

## 一人公司常见自动化场景

| 场景 | 推荐方案 |
|------|----------|
| 定时发推文 | Buffer / Typefully |
| 新用户欢迎邮件 | Mailchimp / Resend |
| 收到付款后发交付物 | Gumroad自带 / Zapier |
| 每日数据报告 | Python脚本 + 邮件 |
| 代码自动部署 | GitHub Actions |
| 备份数据 | Cron + 脚本 |
| 监控网站运行 | UptimeRobot（免费） |
| 客服自动回复 | AI Chatbot |

## 原则
- **先手动跑通，再自动化** — 不要自动化一个你还没搞清楚的流程
- **简单方案优先** — 能用Zapier就不写代码
- **留有人工介入的口子** — 关键环节保留人工确认
