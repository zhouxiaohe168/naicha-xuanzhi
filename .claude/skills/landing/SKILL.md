# LANDING — 落地页快速生成器

一人公司的第一步：让别人看到你在做什么。5分钟出一个能用的页面。

## 当用户说 /landing 时

### Step 1: 确认页面类型

| 类型 | 目的 | 核心元素 |
|------|------|----------|
| **产品介绍页** | 展示产品，吸引用户 | Hero + 功能 + 定价 + CTA |
| **个人主页** | 展示自己，建立品牌 | 自我介绍 + 项目 + 联系方式 |
| **预售/等待页** | 验证需求，收集邮箱 | 痛点 + 方案 + 邮箱收集 |
| **作品集** | 展示过往作品 | 项目列表 + 案例 + 联系方式 |

### Step 2: 收集关键信息
问用户：
1. **一句话说你的产品/服务是什么？**
2. **目标用户是谁？**
3. **用户最大的痛点是什么？**
4. **你的方案怎么解决？**（2-3个核心卖点）
5. **你希望用户做什么？**（注册/购买/联系/订阅）

### Step 3: 生成落地页

使用 **单文件 HTML + Tailwind CDN** 方案（最简单，一个文件搞定）。

**页面结构**：
```
1. Hero区域 — 大标题 + 副标题 + CTA按钮
2. 痛点区域 — 用户的3个痛点
3. 方案区域 — 你的3个核心功能/卖点
4. 社会证明 — 数据/用户评价（可选）
5. 定价区域 — 价格方案（可选）
6. FAQ区域 — 常见问题（可选）
7. 底部CTA — 再次引导行动
8. Footer — 联系方式/社交链接
```

**代码要求**：
- 单文件 HTML，内嵌 Tailwind CDN
- 响应式设计（手机+电脑都好看）
- 深色主题为默认（科技感）
- 平滑滚动 + 简单动画
- 中文内容为主
- 代码中加中文注释，方便用户修改

### Step 4: 部署指引
告诉用户最简单的上线方式：
1. GitHub Pages（免费，推到仓库就上线）
2. Vercel（免费，拖拽上传）
3. Netlify（免费，拖拽上传）

## 设计原则
- **少即是多** — 不要塞太多内容
- **一个页面一个目标** — 只引导用户做一件事
- **手机优先** — 大部分人用手机看
