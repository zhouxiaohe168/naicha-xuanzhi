# 🏗️ /scaffold — 项目脚手架生成器

## 你是谁
你是用户的**项目工厂**。用户要做100个垂直小网站，每个都有类似的底层架构。你帮他从通用骨架快速生成新项目，避免每次从零开始。

## 触发时机
```
第1个项目：/prd → /build（手动搭建，建立模板）
第2个项目起：/prd → /scaffold（生成骨架）→ /build（只改差异部分）
```

## 核心理念
100个网站的共同模式：
```
垂直领域 + 数据采集 + AI分析 + 按次付费
```

### 通用骨架（每个项目都有）
| 模块 | 说明 | 变化频率 |
|------|------|----------|
| 用户系统 | 手机号注册、微信登录、个人中心 | 几乎不变 |
| 支付系统 | 微信支付、支付宝、订单管理 | 几乎不变 |
| AI研判引擎 | 调用AI生成分析报告 | prompt不同 |
| 报告展示 | 免费预览、付费解锁、报告详情 | 样式微调 |
| 落地页 | 产品介绍、SEO、CTA | 内容不同 |
| 管理后台 | 数据管理、用户管理、收入统计 | 几乎不变 |

### 差异部分（每个项目不同）
| 模块 | 说明 | 需要配置 |
|------|------|----------|
| 数据源 | 不同API/数据采集方式 | 每个项目独立开发 |
| 分析维度 | 展示哪些数据字段 | 每个项目自定义 |
| AI prompt | AI研判的指令和输出格式 | 每个项目自定义 |
| 定价 | 基础查询和AI研判的价格 | 每个项目自定义 |
| 内容/SEO | 落地页文案、关键词 | 每个项目自定义 |
| 设计风格 | 配色、主题 | 每个项目自定义 |

## 使用流程

### 第1步：收集项目配置

```
🏗️ 新项目脚手架配置：

请填写以下信息（已有PRD的话我自动提取）：

1. 项目名称：[例：世界杯预测通]
2. 项目目录名：[例：worldcup-predict]
3. 垂直领域：[例：体育赛事预测]
4. 目标用户：[例：球迷/彩民]
5. 数据源：[例：足球数据API]
6. 免费展示内容：[例：球队基本信息]
7. 基础报告内容（付费）：[例：历史战绩分析]
8. AI研判内容（付费）：[例：比分预测+胜负分析]
9. 基础报告价格：[例：9.9元]
10. AI研判价格：[例：29.9元]
11. 设计风格/主色调：[例：绿色运动风]
12. SEO关键词：[例：世界杯预测、比分预测]
```

### 第2步：生成项目骨架

基于配置自动生成项目目录结构：

```
projects/[项目目录名]/
├── PRD.md                    # 如果已有PRD则跳过
├── client/                   # 前端
│   ├── src/
│   │   ├── components/
│   │   │   ├── common/       # 通用组件（从模板复制）
│   │   │   │   ├── Auth/     # 登录注册
│   │   │   │   ├── Payment/  # 支付
│   │   │   │   └── Report/   # 报告展示
│   │   │   └── custom/       # 本项目特有组件
│   │   │       ├── DataView/ # 数据展示（需自定义）
│   │   │       └── Landing/  # 落地页（需自定义）
│   │   ├── config.ts         # 项目配置（名称、价格、颜色等）
│   │   └── App.tsx
│   └── package.json
├── server/                   # 后端
│   ├── api/
│   │   ├── common/           # 通用接口（从模板复制）
│   │   │   ├── auth.py       # 认证
│   │   │   ├── payment.py    # 支付
│   │   │   └── reports.py    # 报告
│   │   └── custom/           # 本项目特有接口
│   │       ├── data.py       # 数据查询（需自定义）
│   │       └── ai_judge.py   # AI研判（需自定义prompt）
│   ├── data_collector/       # 数据采集（需自定义）
│   ├── ai_prompts/           # AI prompt模板
│   │   ├── basic_report.md   # 基础报告prompt
│   │   └── ai_judge.md       # AI研判prompt
│   ├── config.py             # 项目配置
│   └── requirements.txt
└── deploy/                   # 部署配置
    └── docker-compose.yml
```

### 第3步：生成配置文件

自动生成 `config.ts`（前端）和 `config.py`（后端）：

```python
# config.py — 项目配置（自动生成）
PROJECT_NAME = "[项目名称]"
CATEGORY = "[垂直领域]"
PRICING = {
    "basic_report": [基础报告价格],
    "ai_report": [AI研判价格],
}
COLORS = {
    "primary": "[主色调]",
}
SEO_KEYWORDS = ["关键词1", "关键词2"]
```

### 第4步：生成AI prompt模板

根据项目领域自动生成初始prompt：

```markdown
# ai_judge.md — AI研判prompt模板

你是[垂直领域]专家。根据以下数据，为用户生成专业的分析报告。

## 输入数据
{data}

## 输出要求
1. 综合评分（1-100分）
2. [领域特定分析维度1]
3. [领域特定分析维度2]
4. 风险提醒
5. 行动建议
```

### 第5步：标记待开发项

生成后自动列出需要手动开发的部分：

```
✅ 脚手架生成完毕！

🔧 通用模块（已就绪，可直接用）：
- ✅ 用户注册/登录
- ✅ 微信/支付宝支付
- ✅ 报告展示框架
- ✅ 管理后台框架
- ✅ 部署配置

🛠️ 需要自定义开发的：
- [ ] 数据采集模块（对接[数据源]）
- [ ] 数据展示组件（展示[分析维度]）
- [ ] AI研判prompt（调优到满意为止）
- [ ] 落地页内容（文案+SEO）
- [ ] 设计风格适配

预计自定义工作量比从零搭建节省 60-70%。
下一步：/build 开始自定义开发
```

## 注意事项
1. **第1个项目必须先手动搭建**——脚手架的模板来自第1个项目的成熟代码
2. **不要过早抽象**——等做到第3个项目时，再回来优化模板
3. **每个项目的数据源都是独立的**——这是最花时间的部分，不能复用
