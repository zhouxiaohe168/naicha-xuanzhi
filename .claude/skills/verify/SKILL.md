# VERIFY — 完成前验证（灵感来自 Superpowers/verification-before-completion）

**铁律：说"做完了"之前，必须跑一遍验证。**

不验证就说完成 = 自欺欺人。

## 什么时候触发

| 场景 | 自动触发 |
|------|---------|
| 完成一个功能开发 | ✅ |
| 修复一个Bug | ✅ |
| 部署上线之前 | ✅ |
| 用户问"做好了吗" | ✅ |

## 验证清单（按类型）

### 功能开发完成验证

```
□ 1. 代码验证
  ├── 代码能编译/构建通过吗？
  ├── 没有 console.log / debugger 残留？
  └── 没有硬编码的测试数据？

□ 2. 功能验证
  ├── PRD中要求的每个功能点都实现了吗？（逐条对照）
  ├── 正常流程能走通吗？（Happy Path）
  └── 异常情况处理了吗？（空数据、网络错误、无权限）

□ 3. UI验证（如果有界面）
  ├── 手机上显示正常吗？（Playwright截图验证）
  ├── 电脑上显示正常吗？
  └── 交互状态都对吗？（加载中、成功、失败、空状态）

□ 4. 集成验证
  ├── 和其他已有功能冲突吗？
  ├── API接口返回格式对吗？
  └── 数据库操作正确吗？

□ 5. Git验证
  ├── 代码已提交？
  ├── commit message清晰？
  └── 没有意外包含敏感信息？(.env, API key)
```

### Bug修复完成验证

```
□ 1. Bug确实修好了吗？
  └── 用原来的步骤复现 → 确认不再出现

□ 2. 没有引入新Bug吗？
  └── 相关功能都还正常吗？

□ 3. 根本原因解决了吗？
  └── 还是只是治标不治本？
```

### 部署前验证

```
□ 1. 本地环境能正常运行？
□ 2. 环境变量都配好了？
□ 3. 数据库迁移需要跑吗？
□ 4. Playwright自动化测试全部通过？
□ 5. 手机端测试OK？
□ 6. 性能可接受？（页面加载 < 3秒）
```

## 验证结果报告

```markdown
## 验证报告
- 功能：[功能名称]
- 日期：YYYY-MM-DD
- 状态：✅ 全部通过 / ⚠️ 部分通过 / ❌ 未通过

### 通过项
- ✅ 项目1
- ✅ 项目2

### 未通过项（如果有）
- ❌ 项目3 — 原因：XXX — 处理：XXX

### 结论
[可以标记完成 / 需要继续修复]
```

## 原则
- **不放水** — 有一项不通过就不算完成
- **用证据说话** — 截图、测试结果、日志，不是"我觉得OK"
- **Playwright是裁判** — 能用自动化测试的就不靠肉眼
