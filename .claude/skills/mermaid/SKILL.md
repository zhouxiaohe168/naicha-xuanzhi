# Mermaid 图表生成器

将用户的描述转化为 Mermaid 图表代码。支持 22+ 种图表类型，GitHub/VS Code/Obsidian 均可直接渲染。

## 工作流程

1. **理解需求** — 用户想表达什么？
2. **选择图表类型** — 哪种图表最合适？
3. **生成代码** — 输出正确的 Mermaid 语法
4. **输出方式** — 在 markdown 代码块中输出，或写入 `.md` 文件

## 图表类型速查

| 类型 | 关键字 | 适合表达 |
|------|--------|----------|
| **思维导图** | `mindmap` | 发散思考、知识结构、头脑风暴 |
| **流程图** | `flowchart` | 步骤、决策、流程 |
| **时序图** | `sequenceDiagram` | 系统交互、API调用顺序 |
| **类图** | `classDiagram` | 代码结构、对象关系 |
| **状态图** | `stateDiagram-v2` | 状态变化、生命周期 |
| **ER图** | `erDiagram` | 数据库关系 |
| **甘特图** | `gantt` | 项目时间线、进度 |
| **饼图** | `pie` | 比例分布 |
| **时间线** | `timeline` | 历史事件、发展路径 |
| **Git图** | `gitGraph` | 分支策略、版本管理 |
| **用户旅程** | `journey` | 用户体验流程 |
| **象限图** | `quadrantChart` | 优先级矩阵、二维分析 |
| **XY图** | `xychart-beta` | 数据趋势、对比 |
| **框图** | `block-beta` | 系统架构、模块关系 |
| **桑基图** | `sankey-beta` | 流量分布、转化漏斗 |

## 语法关键规则（防止出错）

### 通用规则
1. **特殊字符必须加引号** — 节点文字含 `()[]{}|` 等符号时用双引号包裹
2. **避免 Mermaid 保留字作为节点ID** — 如 `end`, `graph`, `subgraph`, `style`
3. **节点ID只用英文字母和数字** — 显示文字可以用中文
4. **缩进用空格** — 不要用 Tab

### 思维导图 mindmap 规则
```
mindmap
  root((中心主题))
    分支1
      子分支1a
      子分支1b
    分支2
      子分支2a
    分支3
```
- 用缩进表示层级（2个空格）
- `root((...))` 圆形根节点
- `((...))`圆形、`[...]`方形、`(...)` 圆角方形
- 不支持连接线箭头，纯靠缩进

### 流程图 flowchart 规则
```
flowchart TD
    A[开始] --> B{判断条件}
    B -->|是| C[操作1]
    B -->|否| D[操作2]
    C --> E[结束]
    D --> E
```
- `TD` 从上到下，`LR` 从左到右
- `[方框]` `(圆角)` `{菱形}` `((圆形))`
- `-->` 箭头，`---` 无箭头，`-.->` 虚线箭头

### 时间线 timeline 规则
```
timeline
    title 项目发展历程
    2024 : 想法诞生
         : 开始学习
    2025 : 产品上线
         : 获得第一个用户
    2026 : 规模化增长
```

### 饼图 pie 规则
```
pie title 时间分配
    "开发" : 40
    "内容" : 20
    "学习" : 20
    "运营" : 15
    "休息" : 5
```

## 样式建议

- 默认不加复杂样式，保持简洁
- 如果用户要求好看，可以用 `classDef` 和 `style` 自定义颜色
- 中文内容时注意节点宽度，文字不要太长

## 输出格式

始终将 Mermaid 代码放在 markdown 代码块中：

````
```mermaid
[图表代码]
```
````

如果用户要保存为文件，写入 `.md` 文件，GitHub 会自动渲染。
