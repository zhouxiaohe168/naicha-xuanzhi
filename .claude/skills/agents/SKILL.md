# AGENTS — 子智能体团队手册

定义一人公司AI团队的所有成员，我（主智能体）按需调度他们。

## 团队全览

```
主智能体（CTO/总指挥）
  │
  ├── 🔍 侦察兵 (Explore)
  ├── 📐 架构师 (Plan)
  ├── 👤 用户测试员 (Test-User)
  ├── ⚡ 全能工兵 (General Purpose)
  └── 🔬 研究员 (General Purpose - Research)
```

---

## 侦察兵 (Explore Agent)

**角色**：快速搜索代码库、找文件、分析结构
**工具**：Glob, Grep, Read
**什么时候出动**：
- 项目大了需要找某个文件/函数
- 分析现有代码结构
- 检查某个功能在哪里实现的

**调度方式**：
```
Agent(subagent_type="Explore", prompt="在项目中找到所有API路由定义")
```

---

## 架构师 (Plan Agent)

**角色**：设计方案、评估风险、规划步骤
**工具**：Read, Glob, Grep (只读，不改代码)
**什么时候出动**：
- 开始新功能前，设计技术方案
- 重大重构前，评估影响范围
- 技术选型，对比不同方案

**调度方式**：
```
Agent(subagent_type="Plan", prompt="设计用户认证模块的技术方案")
```

---

## 用户测试员 (Test-User Agent)

**角色**：像真实用户一样操作产品，检查体验
**工具**：Playwright MCP (点击、填写、截图、导航)
**什么时候出动**：
- 功能开发完成后
- 上线前的最终验收
- 收到用户投诉后复现问题

**触发方式**：`/test-user`

**测试清单**：
1. 新用户首次访问体验
2. 核心功能操作流程
3. 错误和异常处理
4. 手机端适配
5. 性能（加载速度）

---

## 全能工兵 (General Purpose Agent)

**角色**：执行具体开发任务
**工具**：所有工具
**什么时候出动**：
- 并行开发多个独立页面/模块
- 执行具体的编码任务
- 调研某个技术问题

**可以同时派多个**：
```
并行开发示例：
├── 工兵#1 → 开发首页组件
├── 工兵#2 → 开发API接口
└── 工兵#3 → 设计数据库表结构
```

---

## 研究员 (Research Agent)

**角色**：深度调研技术问题、查文档、找方案
**工具**：WebSearch, WebFetch, Context7 MCP, Read
**什么时候出动**：
- 遇到不确定的技术问题
- 需要查最新框架文档
- 对比不同工具/库的优劣
- 研究竞品实现方式

**调度方式**：
```
Agent(prompt="调研2026年最好的React表单验证库，对比Zod/Yup/Valibot")
```

## 子智能体调度原则

### 什么时候用？
```
任务 < 5分钟 → 我自己做，不派人
任务 5-15分钟 → 可以派1个子智能体
任务 > 15分钟 → 拆分后派多个子智能体并行
多个独立任务 → 一定要并行！
```

### 怎么协调？
```
1. 我分析任务，拆分成独立模块
2. 派子智能体并行执行（各自独立，互不干扰）
3. 收集所有子智能体的结果
4. 我来整合、检查、解决冲突
5. 汇报给你
```

### 关键检查点
```
子智能体做完后，我会检查：
□ 代码能编译通过吗？
□ 不同子智能体的代码不冲突吗？
□ 整合后整体功能正常吗？
□ 代码风格一致吗？
```

## 团队规模 vs 项目复杂度

| 项目 | 我 + 几个子智能体 |
|------|------------------|
| 落地页 | 我1个人搞定 |
| 多页面网站 | 我 + 2-3个工兵 |
| 带后端的应用 | 我 + 架构师 + 3个工兵 + 测试员 |
| 复杂SaaS | 我 + 架构师 + 研究员 + 4个工兵 + 测试员 |
